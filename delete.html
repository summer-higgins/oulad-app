Accepted User Input

include here a description of all data allowed + explanations of accepted input

  • Table Information:
    • id_student

      This is the student ID.
      Accepted values are: numbers.

    • Code Module

      This is the code module.
      The current accepted code modules:

      • AAA
      • BBB
      • CCC
      • DDD
      • EEE
      • FFF
      • GGG

    • Code Presentation:

      This is the code presentation.
      The current accepted code presenations:

      • 2013B
      • 2013J
      • 2014B
      • 2014J

    • id_assessment

      This is the assessment ID corresponding to student interactions in the VLE for a module.
      The current accepted assessment IDs listed in their ranges:

      • 1752 – 1756
      • 1758 – 1762
      • 14984 – 14989
      • 14991 – 15001
      • 15003 – 15013
      • 15015 – 15024
      • 24282 – 24299
      • 25334 – 25368
      • 30709 – 30712
      • 30714 – 30717
      • 30719 – 30722
      • 34860 – 34871
      • 34873 – 34884
      • 34886 – 34897
      • 34899 – 34910
      • 37415 – 37423
      • 37425 – 37433
      • 37435 – 37443

    • Week:

      This is the student's week number by the module start date.
      Instructions on input calculation:

      1. Determine start date of module.
      2. Determine the week you would like to assess by date.
      3. Count the number of weeks from start date to requested week date.
      4. Input value into form.

      Example:
      • Module begins January 1st, 2026.
      • Week requested: February 12th, 2026.
      • User requested to view week 6 of the module.
      • Input: 6

    • Activity Type:

      This is the activity type.
      The current accepted activity types:

      • resource
      • oucontent
      • url
      • homepage
      • subpage
      • glossary
      • oucollaborate
      • forumng
      • dataplus
