Welcome to the OULAD Student Engagement Analytics Dashboard

This dashboard provides a web-based interface for working with student engagement and academic performance data from the OULAD dataset.

Use the navigation menu above to move between the main webpage, data entry forms, update and delete tools, the summary page, and the documentation webpage.

Our Purpose

The purpose of this application is to provide a simple way for users to store, query, and analyze student performance and engagement data from online courses. The app helps users understand how factors such as student background, registration behavior, assessment performance, and VLE activity may influence academic outcomes.

Dataset Information

The dataset used for this project is the Open University Learning Analytics Dataset (OULAD). We chose this dataset because it is publicly available, credible, and already structured in a way that supports relational database storage. It includes linked information about students, courses, assessments, registration, and VLE activity, which makes it well suited for a web-based database application focused on student engagement and academic performance.

Dataset Tables

  • courses - stores module codes, presentation codes, and the length of each module presentation.
  • assessments - stores assessment identifiers, module information, assessment type, due date, and assessment weight.
  • vle - stores information about materials and resources available in the virtual learning environment.
  • studentInfo - stores student demographic information, prior attempts, credits studied, disability status, and final result.
  • studentRegistration - stores module registration and unregistration information for each student.
  • studentAssessment - stores student assessment submissions, scores, and related assessment details.
  • studentClicks - stores total click activity for each student in a module presentation.
  • weeklyClicks - stores click totals organized by student and week.
  • activityTypeClicks - stores click totals organized by student and activity type.
Note: The original OULAD dataset included a very large table named studentVle. For the p was reorganized into the three derived tables studentClicks, weeklyClicks, and activityTypeClicks to make the data easier to store, query, and present in the application.
