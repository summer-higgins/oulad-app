Welcome to the OULAD Student Engagement Analytics Dashboard

This webpage allows users to add a new record to more than one table in the database.

We have provided a layout of instructions and the data form below.

Add Data Description

In this webpage, students are allowed to add data to the OULAD database. The section below includes a description of allowed user input.

Accepted User Input

The descriptions below list all available input. Please read carefully and follow all instructions to ensure accurate data entry.

  • Student Information:
    • Code Module and Code Presentation

      Code Module: This is the code name of the module, which serves as the identifier.
      Code Presentation: This is the code name of the presentation. It consists of the year and "B" for the presentation starting in February and "J" for the presentation starting in October.
      Please follow the format below to submit your module and presentation.
      Users must input the module code and its assigned presentation in their designated submissions, or the form will not accept the input.

      • AAA | 2013J
      • AAA | 2014J
      • BBB | 2013B
      • BBB | 2013J
      • BBB | 2014B
      • BBB | 2014J
      • CCC | 2014B
      • CCC | 2014J
      • DDD | 2013B
      • DDD | 2013J
      • DDD | 2014B
      • DDD | 2014J
      • EEE | 2013J
      • EEE | 2014B
      • EEE | 2014J
      • FFF | 2013B
      • FFF | 2013J
      • FFF | 2014B
      • FFF | 2014J
      • GGG | 2013J
      • GGG | 2014B
      • GGG | 2014J
    • Student ID

      This is the student ID.
      Accepted values are numbers.

    • Gender

      This is the student's gender.
      Accepted values are listed within the form.

    • Region

      This is the student's geographic region during the time they took the module-presentation.
      Accepted values are any valid lettered input.

    • Highest Education

      This is the highest student education level on entry to the module presentation.
      Accepted values are listed within the form.

    • IMD Band

      This is a representation of how wealthy or disadvantaged the area the student lives in is.
      Accepted values are listed within the form.

    • Age Band

      This is the band of the student's age.
      Accepted values are listed within the form.

    • Number of Previous Attempts

      This is the number of times the student has attempted a module.
      Accepted values are listed within the form.

    • Studied Credits

      This is the total number of credits for the module the student is currently studying.
      Accepted values are listed within the form.

    • Disability

      This is an indication of student disability.
      Accepted values are listed within the form.

    • Final Result

      This is the student's final result in the module-presentation.
      Accepted values are listed within the form.

  • Student Registration Information:
    • Registered Date

      This is the student's number of days registered by the module start date.
      Instructions on input calculation:

      1. Determine start date of module.
      2. Determine date registered for module.
      3. Count the number of days from registered date to start date. If the student registered before the start date, use "-" before the value.
      4. Input value into form.

      Example:

      • Module begins January 1st, 2026.
      • Date registered: December 2nd, 2025.
      • User registered for the module 30 days before the start date.
      • Input: -30
    • Unregistered Date

      This is the student's number of days unregistered by the module start date.
      Instructions on input calculation:

      1. Determine start date of module.
      2. Determine date unregistered for module.
      3. Count the number of days from unregistered date to start date. If the student unregistered before the start date, use "-" before the value.
      4. Input value into form.

      Example:

      • Module begins January 1st, 2026.
      • Date unregistered: January 4th, 2026.
      • User unregistered for the module 3 days after the start date.
      • Input: 3
  • Student Assessment Information:

    To accurately input assessment information, please follow all guidelines listed below.

    • Assessment ID

      This is the assessment ID corresponding to student interactions in the VLE for a module.
      The current accepted assessment IDs are listed in these ranges:

      • 1752 – 1756
      • 1758 – 1762
      • 14984 – 14989
      • 14991 – 15001
      • 15003 – 15013
      • 15015 – 15024
      • 24282 – 24299
      • 25334 – 25368
      • 30709 – 30712
      • 30714 – 30717
      • 30719 – 30722
      • 34860 – 34871
      • 34873 – 34884
      • 34886 – 34897
      • 34899 – 34910
      • 37415 – 37423
      • 37425 – 37433
      • 37435 – 37443
    • Date Submitted

      This is the date of student submission for the assessment, measured as the number of days since the start of the module presentation.

      1. Determine start date of module.
      2. Determine date of the assessment submission.
      3. Count the number of days from start date to submission date.
      4. Input value into form.

      Example:

      • Module begins January 1st, 2026.
      • Date submission: January 31st, 2026.
      • User submitted the assessment 30 days after the module start date.
      • Input: 30
    • Banked Flag

      This is a status flag indicating that the assessment result has been transferred from a previous presentation.
      Accepted values are 0 or 1.

    • Assessment Score

      This is the student's assessment score.
      Accepted values are between 0 and 100.
