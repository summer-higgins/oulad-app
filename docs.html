Welcome to the OULAD Student Engagement Analytics Dashboard

This webpage provides documentation for the application, including project information, requirements, setup instructions, and a user guide for each webpage.

It is intended to help users understand the purpose of the application and how the interface is organized.

Project Overview

The purpose of this application is to provide a simple way for users to store, query, and analyze student performance and engagement data from online courses. The app helps users understand how factors such as student background, registration behavior, assessment performance, and VLE activity may influence academic outcomes.

App and Team Information

App Name: OULAD Analytics

Design & Development Team: Elma Okanovic and Summer Higgins

Project Type: Web-based database application interface

Web URL: Local prototype for course submission

System Requirements

Hardware Requirements: Minimum 8 GB RAM, minimum 16 GB available disk space

Software Requirements: PostgreSQL, pgAdmin, local server or browser-based HTML preview, and a modern web browser

Setup Instructions

  1. Install PostgreSQL and pgAdmin on your computer.
  2. Create or restore the OULAD database in PostgreSQL.
  3. Ensure the required tables for the project are available in the database.
  4. Place all HTML files and the stylesheet in one main project folder.
  5. Open the folder in Visual Studio Code or another editor.
  6. Launch the interface locally by opening index.html or by using a local server such as Live Server.

Commands / Instructions for Running the Project

This interface is currently a front-end prototype for the project design stage. It can be viewed locally in one of the following ways:

  • Open index.html directly in a web browser.
  • Open the folder in Visual Studio Code and run the project using a local server extension such as Live Server.
  • Use PostgreSQL and pgAdmin separately to manage the database backend during the implementation stage.

User Guide

Home Page

The main webpage introduces the application, explains the project purpose, and provides information about the OULAD dataset and its tables.

Add Data Page

The add page allows users to enter new values into the database through a form. It also explains the accepted input and how the form is organized.

Delete Data Page

The delete page allows users to remove records from selected tables. To reduce mistakes, the form is organized around primary keys and composite keys.

Update Data Page

The update page allows users to identify an existing record and enter replacement values for the fields that should be changed.

Summary Page

The summary page presents a prototype layout for summarized outputs, charts, and grouped project information that can be generated from the database.

Documentation Page

The documentation page provides setup instructions, system requirements, project information, and a guide to help the user understand how to use the interface.

Note: Database logic, server-side processing, and live query execution will be implemented in the next stage of project.
