Welcome to the OULAD Student Engagement Analytics Dashboard

This webpage allows users to identify an existing record and enter revised values for that record.

This page is part of the interface design and shows how update actions will be organized in the completed database application.

Update Data Description

In this webpage, users are allowed to update existing data in the OULAD database. The user must first identify a specific record by its primary key or composite key, then enter the new value or values that should replace the existing data.

Accepted User Input

The update forms are organized by table. Users should first enter the key fields that identify the record, then enter only the values they want to change.

  • Student Info uses the student and module-presentation record to identify a row.
  • Registration Info updates registration and unregistration day values for a student record.
  • Student Assessment Info updates assessment submission details and score for a selected assessment record.
  • Total Clicks Info updates the stored total click count for a student record.
  • Weekly Clicks Info updates click totals for a specific student and week.
  • Activity Type Clicks Info updates click totals for a specific student and activity type.
Guideline: For this design stage, the forms below demonstrate the intended webpage structure and input organization. Backend update logic will be connected in the implementation stage.
